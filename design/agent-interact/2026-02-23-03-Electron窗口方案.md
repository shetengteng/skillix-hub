# Agent Interact — Electron 独立窗口方案

> 日期：2026-02-23

## 1. 问题

当前 V1 架构基于浏览器 Web Server，存在核心体验问题：

| 问题 | 影响 |
|------|------|
| 用户需要切换到浏览器才能看到弹框 | 打断工作流，用户可能不知道有弹框等待操作 |
| 依赖用户知道 `http://127.0.0.1:7890` | 技术门槛，非开发者难以使用 |
| 浏览器标签页容易被关闭或淹没 | 弹框推送到空气中，Agent 只能超时 |
| 无法置顶显示 | 弹框不够醒目，容易被忽略 |

**期望效果**：Agent 需要用户操作时，一个独立的桌面窗口**直接弹出**在屏幕最前面，用户操作后窗口自动关闭，Agent 继续执行。

## 2. 方案：Electron 独立窗口

### 2.1 核心思路

用 Electron 替代浏览器，将现有 Vue 前端打包为桌面应用。Agent 发起交互时，Electron 窗口自动弹出并置顶，用户操作后窗口关闭。

### 2.2 架构

```
Agent CLI (tool.js dialog '<JSON>')
  │
  ├─ HTTP POST /api/dialog ──→ Node.js Server（保持不变）
  │                              │
  │                              ├─ notification → 立即返回
  │                              └─ 其他类型 → dm.create() 阻塞
  │                                   │
  │                                   └─ broadcast dialog:open
  │                                        │
  │                                        ▼
  │                              Electron BrowserWindow（替代浏览器）
  │                              ┌─────────────────────────┐
  │                              │  置顶 + 居中 + 无边框    │
  │                              │  加载 Vue SPA            │
  │                              │  WebSocket 连接 Server   │
  │                              └─────────────────────────┘
  │                                        │
  │                                        ▼
  │                              用户操作 → ws dialog:response
  │                                        │
  └─ HTTP response ←──────────────────────┘
```

### 2.3 与 V1 的差异

| 维度 | V1（浏览器） | V2（Electron） |
|------|-------------|---------------|
| 窗口类型 | 浏览器标签页 | 独立桌面窗口 |
| 弹出方式 | 需要用户手动切换 | 自动弹出置顶 |
| 关闭行为 | 标签页保持打开 | 操作后自动关闭 |
| 可见性 | 容易被淹没 | 始终在最前面 |
| 安装成本 | 无（浏览器自带） | 需要 Electron 依赖 |
| 包大小 | 0 | ~150MB（Electron 运行时） |

### 2.4 窗口行为设计

| 行为 | 说明 |
|------|------|
| 弹出时机 | 收到 `dialog:open` 事件时自动创建窗口 |
| 窗口位置 | 屏幕居中 |
| 置顶 | `alwaysOnTop: true`，确保用户能看到 |
| 窗口大小 | 根据交互类型自适应（confirm: 480×400, chart: 720×560, form: 520×600） |
| 关闭时机 | 用户操作完成后自动关闭（发送 response 后） |
| 多弹框 | 新弹框创建新窗口，旧窗口保持 |
| 无操作 | 超时后窗口自动关闭 |
| 样式 | 无系统标题栏（frameless），自定义标题区域可拖拽 |

### 2.5 notification 特殊处理

notification 类型不弹窗口，改用 Electron 的 `Notification` API 发送系统原生通知：

```javascript
const { Notification } = require('electron');
new Notification({ title, body: message, silent: false }).show();
```

## 3. 技术方案

### 3.1 目录结构

```
skills/agent-interact/
├── tool.js                    # CLI 入口（不变）
├── lib/
│   ├── server.js              # HTTP + WS 服务（不变）
│   ├── dialog-manager.js      # 弹框管理（不变）
│   └── config.js              # 配置（不变）
├── electron/
│   ├── main.js                # Electron 主进程
│   ├── preload.js             # 预加载脚本（安全桥接）
│   └── package.json           # Electron 依赖
└── ui/                        # Vue 前端（不变，Electron 直接加载）
    └── dist/
        └── index.html
```

### 3.2 Electron 主进程（electron/main.js）

核心职责：
1. 连接 WebSocket Server
2. 收到 `dialog:open` → 创建 BrowserWindow
3. 收到 `dialog:close` → 关闭对应窗口
4. notification 类型 → 发送系统通知

```javascript
// 伪代码
const { app, BrowserWindow, Notification } = require('electron');
const WebSocket = require('ws');

const windows = new Map(); // dialogId → BrowserWindow

function connectWs(port) {
  const ws = new WebSocket(`ws://127.0.0.1:${port}/ws`);

  ws.on('message', (raw) => {
    const msg = JSON.parse(raw);

    if (msg.event === 'dialog:open') {
      const dialog = msg.data;

      if (dialog.type === 'notification') {
        new Notification({
          title: dialog.title,
          body: dialog.message,
        }).show();
        return;
      }

      const size = getWindowSize(dialog.type);
      const win = new BrowserWindow({
        width: size.width,
        height: size.height,
        alwaysOnTop: true,
        center: true,
        frame: false,
        resizable: false,
        webPreferences: {
          preload: path.join(__dirname, 'preload.js'),
        },
      });

      // 加载 Vue SPA，传递 dialogId 参数
      win.loadURL(`http://127.0.0.1:${port}?dialogId=${dialog.id}`);
      windows.set(dialog.id, win);
    }

    if (msg.event === 'dialog:close') {
      const win = windows.get(msg.data.id);
      if (win && !win.isDestroyed()) win.close();
      windows.delete(msg.data.id);
    }
  });
}

function getWindowSize(type) {
  const sizes = {
    confirm: { width: 480, height: 400 },
    wait: { width: 440, height: 320 },
    chart: { width: 720, height: 560 },
    form: { width: 520, height: 600 },
    approval: { width: 520, height: 480 },
    progress: { width: 500, height: 440 },
  };
  return sizes[type] || { width: 500, height: 400 };
}
```

### 3.3 启动流程变更

```
tool.js start
  │
  ├─ 启动 Node.js Server（不变）
  └─ 启动 Electron 进程（新增）
       └─ Electron 连接 WS，等待弹框事件

tool.js dialog '<JSON>'
  │
  ├─ 检查服务是否运行（不变）
  ├─ 检查 Electron 是否运行（新增）
  │   └─ 未运行 → 自动启动 Electron
  └─ POST /api/dialog（不变）
       └─ Server broadcast → Electron 收到 → 弹窗
```

### 3.4 前端适配

Vue 前端需要少量适配：

1. **单弹框模式**：Electron 窗口只显示一个弹框（通过 URL 参数 `dialogId` 过滤）
2. **无边框拖拽**：标题区域添加 `-webkit-app-region: drag` 支持窗口拖拽
3. **关闭后通知主进程**：用户操作后通过 `window.close()` 关闭窗口

### 3.5 Fallback 策略

Electron 不可用时（未安装、启动失败），自动 fallback 到浏览器模式：

```
dialog 命令
  │
  ├─ 尝试启动 Electron
  │   ├─ 成功 → Electron 窗口模式
  │   └─ 失败 → Fallback 到浏览器模式
  │              ├─ 发送系统通知
  │              └─ 打开浏览器
  └─ POST /api/dialog
```

## 4. 实现计划

### Phase 1：Electron 基础集成

- 添加 Electron 依赖
- 实现 `electron/main.js`（WS 连接 + 窗口创建）
- `tool.js start` 同时启动 Electron
- confirm 类型验证

### Phase 2：完整类型支持

- 所有 7 种交互类型的窗口大小适配
- notification → 系统原生通知
- 前端单弹框模式适配
- 无边框窗口拖拽

### Phase 3：体验优化

- Fallback 到浏览器模式
- Electron 进程管理（PID、健康检查）
- 窗口动画（弹出/关闭）
- 多显示器支持（弹在活跃显示器上）

## 5. 风险与权衡

| 风险 | 影响 | 缓解 |
|------|------|------|
| Electron 包体积大（~150MB） | 首次安装慢 | 作为可选依赖，不强制安装 |
| Electron 进程常驻内存 | 资源占用 | 空闲时自动退出，需要时再启动 |
| 跨平台兼容 | macOS/Windows/Linux 行为差异 | 优先 macOS，其他平台渐进支持 |
| 与浏览器模式并存 | 维护两套启动逻辑 | Fallback 策略，Electron 优先 |

## 6. 替代方案对比

| 方案 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| Electron 窗口 | 置顶弹出、原生体验、功能完整 | 包大、资源占用 | 需要完整 UI 交互 |
| macOS osascript 原生对话框 | 零依赖、最轻量 | 只支持简单选择，无法展示图表 | 仅 confirm/approval |
| Tauri | 包小（~10MB）、性能好 | 需要 Rust 工具链 | 长期方案 |
| VS Code Webview | 在 IDE 内弹出 | 绑定 VS Code/Cursor | 仅 Cursor 用户 |
