# Agent Interact 技术文档

> 日期：2026-02-23
> 版本：V2（7 种交互类型）

## 1. 概述

Agent Interact 采用 Local Web Server 架构，后端为 Node.js（Express + ws），前端为 Vue 3 + shadcn-vue SPA。Agent 通过 CLI 或 HTTP API 发起交互请求，后端通过 WebSocket 推送到浏览器前端，用户操作后响应回传给 Agent。

## 2. 后端代码结构

```
skills/agent-interact/
├── tool.js                        # CLI 入口
│   - start                        # 启动后台服务（spawn detached 子进程）
│   - stop                         # 停止服务（读取 PID 文件 kill 进程）
│   - status                       # 检查服务状态（HTTP GET /api/status）
│   - dialog '<JSON>'              # 发送弹框（HTTP POST /api/dialog，阻塞等待结果）
│
├── package.json                   # 依赖：express, ws
│
└── lib/
    ├── config.js                  # 常量配置
    │   - DEFAULT_PORT: 7890       # 默认监听端口
    │   - DEFAULT_TIMEOUT: 60      # 弹框默认超时（秒）
    │   - MAX_TIMEOUT: 600         # 最大超时（秒）
    │   - PID_FILE: .server.pid    # 进程 PID 文件路径
    │   - UI_DIST: ui/dist/        # 前端构建产物目录
    │
    ├── server.js                  # HTTP + WebSocket 服务
    │   - createServer(port)       # 创建 Express app + HTTP server + WebSocket server
    │   - POST /api/dialog         # 创建弹框（阻塞等待响应）
    │     - 验证 type 是否在 7 种有效类型中
    │     - notification 类型走 dm.notify() 非阻塞路径
    │     - 其他类型走 dm.create() 阻塞路径
    │   - GET /api/status          # 返回 { status, port, activeDialogs }
    │   - GET /api/dialogs         # 返回活跃弹框列表
    │   - DELETE /api/dialog/:id   # 取消指定弹框
    │   - WebSocket /ws            # 前端实时通信端点
    │   - express.static(UI_DIST)  # 静态文件 serve 前端 SPA
    │
    ├── dialog-manager.js          # 弹框生命周期管理（核心）
    │   - create(request) → Promise  # 创建弹框，返回 Promise（阻塞直到用户响应或超时）
    │   - notify(request) → string   # 推送通知（非阻塞，立即返回 dialogId）
    │   - respond(id, result)        # 处理用户响应，resolve 对应 Promise
    │   - cancel(id)                 # 取消弹框（action: __cancelled）
    │   - list() → Array             # 列出所有未 resolve 的弹框
    │   - shutdown()                 # 关闭所有弹框（action: __shutdown）
    │   - addClient(ws)              # 注册 WebSocket 客户端，补发活跃弹框
    │   - removeClient(ws)           # 移除 WebSocket 客户端
    │   - _broadcast(message)        # 向所有连接的客户端广播 JSON 消息
    │
    └── response.js                # 标准 JSON 输出工具
        - success(data)            # → { result: data, error: null }
        - error(message)           # → { result: null, error: message }
```

## 4. 数据流

```
Agent CLI (tool.js dialog '<JSON>')
  │
  ├─ HTTP POST /api/dialog ──→ server.js
  │                              │
  │                              ├─ type === 'notification'?
  │                              │   ├─ YES → dm.notify() → broadcast → 立即返回 { dialogId, action: 'delivered' }
  │                              │   └─ NO  → dm.create() → 返回 Promise（阻塞）
  │                              │
  │                              └─ dm.create():
  │                                   ├─ 生成 dialogId (d-xxxxxxxxxxxx)
  │                                   ├─ 设置超时定时器
  │                                   ├─ broadcast { event: 'dialog:open', data: { id, ...request } }
  │                                   └─ 等待 resolve...
  │
  │  WebSocket /ws ──→ ws-client.ts
  │                      ├─ dialog:open → dialogs.push(data)
  │                      │                 → DialogContainer 渲染对应组件
  │                      │                   → 用户操作（点击/填写/选择）
  │                      │                     → respond(id, action, data)
  │                      │                       → ws.send({ event: 'dialog:response', data: { id, action, data } })
  │                      │
  │                      └─ dialog:close → dialogs 移除对应 id
  │
  │  server.js WebSocket onmessage
  │    → dm.respond(id, { action, data })
  │      → clearTimeout
  │      → broadcast dialog:close
  │      → resolve Promise
  │
  └─ HTTP response ──→ tool.js stdout JSON
```

## 5. 配置项

| 参数 | 默认值 | 说明 |
|------|--------|------|
| DEFAULT_PORT | 7890 | 服务监听端口 |
| DEFAULT_TIMEOUT | 60 | 弹框默认超时（秒） |
| MAX_TIMEOUT | 600 | 弹框最大超时（秒） |
| PID_FILE | .server.pid | 服务进程 PID 文件 |
| UI_DIST | ui/dist/ | 前端构建产物目录 |

## 6. 支持的交互类型

| 类型 | 阻塞 | 用途 | 响应 action |
|------|------|------|-------------|
| confirm | 是 | 用户从选项中选择 | 选项 id |
| wait | 是 | 等待用户完成外部操作 | confirmed / __cancelled |
| chart | 是 | 图表数据可视化 | 按钮 id |
| notification | 否 | 推送通知提醒 | delivered（立即返回） |
| form | 是 | 收集用户结构化输入 | submitted（附带 formData） |
| approval | 是 | 敏感操作审批门控 | approved / rejected |
| progress | 是 | 展示多步骤任务进度 | 按钮 id |

## 7. 扩展指南

### 添加新交互类型

1. **后端**：
   - `lib/server.js` → `validTypes` 数组添加新类型名
   - 如果是非阻塞类型，在 `POST /api/dialog` 中添加分支逻辑
   - 如果需要特殊生命周期，在 `lib/dialog-manager.js` 中添加方法

2. **前端**：
   - `src/lib/types.ts` → 添加新接口（extends BaseDialog）
   - `src/lib/types.ts` → DialogData 联合类型添加新接口
   - `src/components/` → 创建新 Vue 组件
   - `src/components/DialogContainer.vue` → 添加 `v-else-if` 分支

3. **文档**：
   - `SKILL.md` → 添加类型说明和 CLI 示例
   - 本技术文档 → 更新类型表格

### 添加新 shadcn-vue 组件

```bash
cd skills/agent-interact/ui
npx shadcn-vue@latest add <component-name> -y
```

### 前端开发模式

```bash
cd skills/agent-interact/ui
npm run dev
```

Vite 开发服务器会将 `/api` 和 `/ws` 代理到 `localhost:7890`，需要先启动后端服务。
