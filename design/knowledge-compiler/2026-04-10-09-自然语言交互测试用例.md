# Knowledge Compiler 自然语言交互测试用例

> 日期：2026-04-10
> 目的：验证 Knowledge Compiler 在真实对话中的可用性，用户通过自然语言与 AI Agent 交互，Agent 自动识别意图并调用 kc 命令

## 前提条件

- 已安装 typer 和 pyyaml（`pip3 install typer pyyaml`）
- `skills/knowledge-compiler/SKILL.md` 已被 Agent 识别
- 测试目录为干净的临时目录（避免干扰）

## 环境准备

```bash
SKILL_PATH="skills/knowledge-compiler"
# 如果是临时测试目录
mkdir -p /tmp/kc-test && cd /tmp/kc-test
```

## 如何判断效果

### 效果评估维度

| 维度 | 指标 | 判断标准 |
|------|------|---------|
| **命令可用性** | 7 个命令全部可执行 | init/add/compile/query/lint/status/browse 无报错 |
| **编译产物质量** | 概念文章结构完整 | 有 frontmatter、coverage 标记、source 引用、section 结构 |
| **增量能力** | 变更检测准确 | 修改 1 个文件 → 只重编译受影响概念，未变化概念不动 |
| **质量检查** | 门禁检出已知问题 | lint 能检出 stub 的低覆盖、空 section、aging 等 |
| **状态准确** | status 数字一致 | source files 数 = raw/ 下文件数，concepts 数 = wiki/concepts/ 下文件数 |
| **分类正确** | browse 分组合理 | schema 定义的分类与 browse 输出一致 |
| **查询有效** | query 返回相关结果 | 问"API gateway"→ 返回含 gateway 关键词的概念 |
| **编辑保护** | 手动编辑不丢失 | 编辑正文 → 重编译 → 正文不变，只有 frontmatter 更新 |

### 快速效果检验流程（5 分钟）

```bash
SKILL_PATH="skills/knowledge-compiler"
cd /tmp && rm -rf kc-e2e && mkdir kc-e2e && cd kc-e2e

# 1. 初始化
python3 $SKILL_PATH/main.py init
# ✅ 判断：raw/ 和 wiki/ 目录存在

# 2. 准备测试文件
echo '# Cache Design\n\nWe use Redis for caching.\n\n## Strategy\nCache-aside pattern.' > raw/designs/cache.md
echo '# Auth Design\n\nJWT-based authentication.\n\n## Flow\nToken refresh flow.' > raw/designs/auth.md

# 3. 编译
python3 $SKILL_PATH/main.py compile
# ✅ 判断：wiki/concepts/ 下生成了 2 个 .md 文件
# ✅ 判断：schema.md Uncategorized 下有 2 个 slug
# ✅ 判断：INDEX.md 有概念表格

# 4. 增量验证
echo 'Updated content.' >> raw/designs/cache.md
python3 $SKILL_PATH/main.py compile
# ✅ 判断：Phase 1 显示 "1 changed"（不是 2 new）

# 5. 编辑保护验证
echo '\n## My Notes\nThis is my manual note.' >> wiki/concepts/cache-design.md
python3 $SKILL_PATH/main.py compile
cat wiki/concepts/cache-design.md | grep "My Notes"
# ✅ 判断：grep 能找到 "My Notes"，手动编辑被保留

# 6. 质量检查
python3 $SKILL_PATH/main.py lint
# ✅ 判断：输出包含 Hard Gates / Soft Gates / 概念状态

# 7. 状态查看
python3 $SKILL_PATH/main.py status
# ✅ 判断：Source files = 2, Concepts = 2

# 8. 浏览
python3 $SKILL_PATH/main.py browse
# ✅ 判断：列出 2 个概念

# 9. 查询
python3 $SKILL_PATH/main.py query "cache"
# ✅ 判断：返回 cache-design 概念
```

### 已知局限

| 局限 | 原因 | 预期行为 |
|------|------|---------|
| 概念文章是 stub（低覆盖） | 当前 compile 生成骨架 + prompt，需 AI Agent 执行精细编译 | lint 会报 low_coverage 和 nonempty_content 失败 |
| 分类全在 Uncategorized | classifier 按标题推断 slug，未接入 schema 约束 | 用户可手动在 schema.md 中移动概念到正确分类 |
| query 基于关键词匹配 | 无语义搜索 | 关键词不匹配时可能漏掉相关概念 |
| 关系/交叉引用为空 | 需要 AI 编译时自动抽取 | `## Related` 和 frontmatter `relations` 暂为空 |

## 测试注意事项

1. **Agent 触发**：确保 SKILL.md 中的 triggers 关键词正确（知识编译、knowledge compiler、kc、wiki compile、知识库、编译知识）
2. **路径**：所有命令需使用 `python3 $SKILL_PATH/main.py` 完整路径
3. **编码**：确保终端支持 UTF-8（中文输出）
4. **清理**：每个测试开始前可用 `rm -rf raw/ wiki/ .kc-config.json .compile-state.json` 清理
5. **stub 与 AI 编译**：当前 `compile` 生成的是 stub 文章 + prompt，AI 精细编译需要 Agent 执行 `.kc-compile-prompt.md`
