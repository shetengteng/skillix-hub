# Knowledge Compiler — 自然语言使用示例

> 日期：2026-04-10
> 本文档展示用户如何通过自然语言与 Knowledge Compiler Skill 交互

---

## 场景 1：初始化知识库

**用户说：**
> 帮我初始化一个知识库

**AI 执行流程：**
1. 读取 SKILL.md → 匹配 "初始化知识库" → 路由到 `commands/kc-init.md`
2. 读取 `kc-init.md` 指令
3. 检查当前目录是否已有 `.kc-config.json`
4. 扫描现有 `.md` 文件，询问用户选择源目录
5. 创建 `raw/` + `wiki/` 目录结构
6. 生成 `.kc-config.json`
7. 创建 wiki/INDEX.md 和 wiki/log.md 脚手架

**AI 输出：**
```
Knowledge base initialized.

Sources: raw/designs/, raw/decisions/, raw/research/, raw/notes/
Output:  wiki/
Mode:    recommended

Next steps:
  1. Add materials to raw/ subdirectories
  2. Run: kc compile
```

## 场景 3：编译知识库

**用户说：**
> 编译一下

**AI 执行流程：**
1. 读取 SKILL.md → 匹配 "编译" → 路由到 `commands/kc-compile.md`
2. 读取 `kc-compile.md` → 调用 `skills/wiki-compiler.md` 5 阶段管道
3. **Phase 1 Scan**：读取 `.compile-state.json`，对比 raw/ 文件 mtime
   - 发现 3 个新文件、1 个修改文件
4. **Phase 2 Classify**：读取每个文件标题和前 500 字
   - 推断出 2 个 topic slug：`api-gateway-design`、`event-driven-architecture`
5. **Phase 3 Compile**：读取源文件全文
   - 生成 `wiki/concepts/api-gateway-design.md`（使用 `templates/article.md` 模板）
   - 更新 `wiki/concepts/event-driven-architecture.md`
   - 标记覆盖度：Summary=high, Key Decisions=medium, Gotchas=low
6. **Phase 3.5 Schema**：更新 `wiki/schema.md`
7. **Phase 4 Index + Verify**：
   - 更新 `wiki/INDEX.md`
   - Hard Gate：全部通过
   - Soft Gate：1 个孤立概念警告
8. **Phase 5 State**：更新 `.compile-state.json` + 追加 `wiki/log.md`

**AI 输出：**
```
Compiled: 1 new topic, 1 updated, 5 unchanged
Hard Gates: all passed
Soft Gates: 1 warning — wiki/concepts/old-topic.md has no inbound links
Wiki: wiki/INDEX.md
```

## 场景 4：查询知识

**用户说：**
> API 网关的设计取舍是什么？

**AI 执行流程：**
1. 读取 SKILL.md → 匹配知识查询意图 → 路由到 `commands/kc-query.md`
2. 读取 `wiki/INDEX.md` → 识别相关主题 `api-gateway-design`
3. 读取 `wiki/concepts/api-gateway-design.md`
4. 检查 Key Decisions 章节覆盖度为 `medium` → 引用但标注不确定性
5. 检查 Gotchas 章节覆盖度为 `low` → 回查 `raw/designs/api-gateway-design.md` 获取更多细节

**AI 输出：**
> 基于 Wiki 和原始设计文档，API 网关的核心设计取舍包括：
>
> 1. **集中式 vs 分布式网关**：选择了集中式方案... [wiki: api-gateway-design]
> 2. **认证策略**：在网关层做 JWT 验证... [source: raw/designs/api-gateway-design.md]
>
> *注：Gotchas 章节覆盖度较低，以上信息已从原始文档补充验证。*
