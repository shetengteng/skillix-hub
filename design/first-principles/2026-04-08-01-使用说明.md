# First Principles Skill 使用说明

## 概述

First Principles（第一性原理）是一个推理分析工具。将问题拆解到不可置疑的基础事实（物理定律 / 逻辑必然 / 实证数据），系统挑战每一个假设，再从地基向上重建解决方案——而非依赖惯例或类比。

## 核心能力

| 能力 | 说明 |
|------|------|
| 假设拆解 | 将问题分解为显式假设和隐式假设 |
| 地基检验 | 对每个假设执行"物理定律 / 逻辑必然 / 实证事实 vs 惯例"检验 |
| 伪约束识别 | 标记并消除不必要的继承约束，打开新设计空间 |
| 迭代重建 | 仅基于已验证的地基事实逐步构建解决方案 |
| 复杂度路由 | 自动判断 T1-T4 复杂度，简单问题直接回答，复杂问题完整分析 |

## 适用场景

| 场景 | 触发信号示例 |
|------|-------------|
| 🏗️ 架构决策 | "为什么用这个架构"、"重新设计系统" |
| 💰 成本质疑 | "为什么这么贵"、"成本能不能降" |
| 🔧 根因分析 | "这个 bug 为什么反复出现"、"性能瓶颈在哪" |
| 📋 产品定义 | "用户真正需要什么"、"这个需求合理吗" |
| 🔬 技术选型 | "用 A 还是 B"、"为什么选这个框架" |
| 🧭 战略决策 | "要不要做这件事"、"Build vs Buy" |

## 分析三阶段

```
Phase 1 — 拆解：列举所有假设（显式 + 隐式）
    ↓
Phase 2 — 地基事实：逐一挑战，标记 ✅ 地基 / ⚠️ 待验证 / ❌ 伪约束
    ↓
Phase 3 — 重建：仅基于地基事实构建方案，标注取舍和崩溃点
    ↑                                                          │
    └──── 迭代：重建中发现新假设 → 回到 Phase 2 ──────────────┘
```

## 使用示例

### 示例 1：架构选型

```
用户: 我们打算用微服务架构重构单体应用，你觉得怎么样？

AI (T3 实质性分析):
  Phase 1 — 拆解:
    - 显式假设：微服务能提升开发效率
    - 隐式假设：团队有足够的 DevOps 能力、网络延迟可忽略、数据一致性可通过最终一致解决

  Phase 2 — 地基检验:
    | 假设                    | 挑战                          | 判定         |
    | 微服务提升开发效率       | 取决于团队规模和服务边界划分    | ⚠️ 需要验证  |
    | 团队有 DevOps 能力       | 当前是否有 K8s/监控/链路追踪   | ⚠️ 需要验证  |
    | 单体已成为瓶颈           | 是部署频率还是性能瓶颈？       | ❌ 伪约束    |

  Phase 3 — 重建:
    推荐：先模块化单体（Modular Monolith），验证服务边界后再逐步拆分
    取舍：牺牲独立部署能力，换取较低的运维复杂度
```

### 示例 2：成本挑战

```
用户: 我们的云服务账单每月 5 万，能不能降下来？

AI (T3 成本/约束分析):
  物料分解: [计算: 2.1万] + [存储: 1.5万] + [网络: 0.8万] + [其他: 0.6万]
  与声称成本的差距: 存储中 60% 是未清理的旧备份（伪约束）
  接近下限的路径: 清理备份 + Reserved Instance → 预计降至 3.2 万/月
```

### 示例 3：根因分析

```
用户: 部署后 API 响应偶尔超时，重启就好了

AI (T3 根因分析):
  5-Whys:
    Why 1: 为什么超时？→ 连接池耗尽
    Why 2: 为什么耗尽？→ 慢查询占用连接不释放
    Why 3: 为什么有慢查询？→ 缺失索引 + 数据量增长
    Why 4: 为什么重启能恢复？→ 连接池重置，但根因未解决
    根本假设: "数据库查询性能不会随数据增长退化"

  根因修复: 添加索引 + 连接池超时配置
  症状修复（不推荐）: 定时重启服务
```

## 与 Socratic Skill 的关系

两个 Skill 互补，不竞争：

| Skill | 定位 | 输出 |
|-------|------|------|
| **socratic** | 清理地基——追问和挑战，暴露矛盾 | 更清晰的定义 |
| **first-principles** | 从地基重建——拆解到基岩事实再构建 | 具体方案 |

**推荐使用顺序**：

```
/socratic         →  明确问题、暴露伪约束
/first-principles →  挑战约束、从基岩重建方案
```

## 复杂度分级

| 层级 | 信号 | Agent 动作 |
|------|------|-----------|
| T1 简单 | 常规决策，领域清晰 | 直接回答 |
| T2 局部 | 单个假设需要挑战 | 快速挑战 + 修复 |
| T3 实质性 | 多个继承假设，非平凡成本/风险 | 完整三阶段分析 |
| T4 架构级 | 系统级决策，长期影响 | 完整分析 + 替代方案 + 现状调研 |

## 触发词

- **第一性原理**：用第一性原理分析、从基础重新思考
- **假设挑战**：为什么一定要用X、能不能不做Y、挑战这个假设
- **成本质疑**：为什么这么贵、成本能不能降、钱花在哪了
- **根因分析**：为什么反复出现、根本原因是什么
- **重新设计**：打破惯例、重新设计、从零开始
