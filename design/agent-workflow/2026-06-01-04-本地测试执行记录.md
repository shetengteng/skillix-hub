## Test 4: Mock 端到端闭环

**workflow 文件** `/tmp/aw-smoke/mock-e2e.yaml`：
```yaml
name: mock-e2e-test
nodes:
  - alias: ping
    type: agent_call
    executor: mock
    prompt: "Say hello"
    output: pong
  - alias: echo
    type: agent_call
    executor: mock
    prompt: "Echo back: {{pong}}"
    output: final
```

**执行命令**：
```bash
AGENT_WORKFLOW_ENABLE_MOCK=1 \
AGENT_WORKFLOW_MOCK_PING="hello world" \
AGENT_WORKFLOW_MOCK_ECHO="echoed: hello world" \
agent-workflow start '{"workflow":"/tmp/aw-smoke/mock-e2e.yaml","caller":"cursor-test"}'
```

**输出**：
```json
{
  "result": {
    "action": "completed",
    "run_id": "wf-20260601-025821-3cd138",
    "status": "completed",
    "vars": {
      "pong": "hello world",
      "final": "echoed: hello world"
    }
  }
}
```

**验证点**：
- 两节点自动 chain 完成（mock 不需要 caller 循环）
- `vars.pong` = ping 节点输出，`vars.final` = echo 节点引用 `{{pong}}` 后输出
- 变量传递链路正确

## 踩坑记录：Cursor Shell 调 OpenCode

在 Cursor IDE 内的 Shell 调 `opencode run` 时，因为 Cursor Shell 不提供 TTY，OpenCode 的 TUI 渲染会失效导致输出丢失。

**问题现象**：
```bash
# 以下命令在 Cursor Shell 中执行，看不到输出
opencode run --dangerously-skip-permissions "查询 xxx"
opencode run ... 2>&1 | head -100   # 也不行
```

**解决方案**：
```bash
# 方案 1：nohup + json 格式 + 文件重定向（推荐）
nohup opencode run --format json -m "ai_chat/claude-sonnet-4-6" \
  --dangerously-skip-permissions "你的指令" \
  > /tmp/opencode-output.txt 2>&1 &

# 等待完成后解析
python3 -c "
import json
with open('/tmp/opencode-output.txt') as f:
    for line in f:
        line = line.strip()
        if not line: continue
        try:
            ev = json.loads(line)
            if ev.get('type') == 'tool_use':
                tool = ev['part']['tool']
                output = ev['part']['state']['output']
                print(f'Tool: {tool}')
                print(f'Output: {output[:500]}')
                print('---')
        except: pass
"
```

**关键参数**：
- `--format json` — 绕过 TUI，输出 JSONL 事件流
- `-m "ai_chat/claude-sonnet-4-6"` — 指定模型
- `--dangerously-skip-permissions` — 跳过权限确认（测试用）

### 辅助脚本 `run-opencode.sh`

已将上述 workaround 封装为 `skills/agent-workflow/run-opencode.sh`，一行命令即可在 Cursor Shell 中跑 OpenCode：

```bash
bash skills/agent-workflow/run-opencode.sh "你的自然语言指令"
bash skills/agent-workflow/run-opencode.sh "指令" --model "ai_chat/claude-sonnet-4-6"
bash skills/agent-workflow/run-opencode.sh "指令" --raw       # 原始 JSONL
bash skills/agent-workflow/run-opencode.sh "指令" --timeout 120
```

输出分三段：Skills Loaded → Tool Calls → Final Response。
