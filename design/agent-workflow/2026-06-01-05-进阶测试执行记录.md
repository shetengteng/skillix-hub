## Test 6: wait_user 暂停/恢复

### start（停在 wait_user）
```bash
AGENT_WORKFLOW_ENABLE_MOCK=1 \
AGENT_WORKFLOW_MOCK_GENERATE="这是AI生成的文章草稿" \
AGENT_WORKFLOW_MOCK_FINALIZE="最终确认完毕" \
agent-workflow start '{"workflow":"/tmp/aw-smoke/wait-user-test.yaml","caller":"manual-test"}'
```

**输出要点**：
- `action: "wait_user"`, `status: "waiting_user"`
- `payload.message`: "生成的内容是：这是AI生成的文章草稿\n你是否采纳？"
- `payload.schema`: `{type: object, required: [approved], properties: {approved: {type: boolean}, comment: {type: string}}}`

### resume（用户确认）
```bash
AGENT_WORKFLOW_ENABLE_MOCK=1 \
AGENT_WORKFLOW_MOCK_FINALIZE="最终确认完毕" \
agent-workflow resume '{"run_id":"<run_id>","input":{"approved":true,"comment":"写得不错"}}'
```

**输出要点**：
- `status: "completed"`
- `vars.content`: "这是AI生成的文章草稿"
- `vars.confirm`: `{approved: true, comment: "写得不错"}`
- `vars.summary`: "最终确认完毕"

**发现的文档 Bug**：SKILL.md 中 resume 参数写的是 `user_input`，实际 CLI 接受的是 `input`。已修复。

## Test 7: Schema 校验错误

### 错误输入（缺少 required 的 approved 字段）
```bash
agent-workflow resume '{"run_id":"<run_id>","input":{"comment":"只填了评论"}}'
```

**输出**：
```json
{
  "error": {
    "code": "SCHEMA_VIOLATION",
    "message": "wait_user input does not match schema: 'approved' is a required property",
    "retryable": true
  }
}
```

### 修正后重试
```bash
agent-workflow resume '{"run_id":"<run_id>","input":{"approved":false,"comment":"需要修改"}}'
```

**输出**：`status: "completed"` — workflow 没有进入终态，retryable 后修正成功。

## Test 8: Loop 循环

### start → 进入 loop body 的 wait_user
```bash
AGENT_WORKFLOW_ENABLE_MOCK=1 \
AGENT_WORKFLOW_MOCK_FIRST_DRAFT="AI安全初稿v1" \
AGENT_WORKFLOW_MOCK_REFINE="AI安全修订版" \
agent-workflow start '{"workflow":"/tmp/aw-smoke/loop-test.yaml","caller":"manual-test"}'
```

### 3 轮循环
| 轮次 | 用户输入 | mock refine 输出 | 结果 |
|---|---|---|---|
| R1 | `{approved:false, feedback:"需要更详细"}` | AI安全修订v2 | wait_user（继续） |
| R2 | `{approved:false, feedback:"还差一点"}` | AI安全修订v3 | wait_user（继续） |
| R3 | `{approved:true, feedback:""}` | AI安全终稿 | **completed** |

**最终 vars**：`{topic: "AI安全", draft: "AI安全终稿", review: {approved: true, feedback: ""}}`

**关键发现**：loop body 是 review → refine，即使 approve 后 refine 仍会执行（do-while 语义：先跑完 body 再检查 condition）。因此每轮 resume 都需要带 `AGENT_WORKFLOW_MOCK_REFINE` 环境变量。
