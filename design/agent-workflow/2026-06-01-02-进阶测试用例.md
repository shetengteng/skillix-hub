## 测试 6：用户阻塞节点 — 暂停等人确认再继续

**场景故事**：你启动了一个 workflow，它先让 mock 产出一段文字，然后停下来等你确认——你给出确认后它才继续。

**workflow 文件** `/tmp/aw-smoke/wait-user-test.yaml`：

```yaml
name: wait-user-test
version: "1.0"
description: 测试 wait_user 节点的暂停与恢复

nodes:
  - alias: generate
    type: agent_call
    executor: mock
    prompt: "生成一段文字"
    output: content

  - alias: confirm
    type: wait_user
    description: 让用户确认是否采纳
    message: "生成的内容是：{{content}}\n你是否采纳？"
    schema:
      type: object
      required: [approved]
      properties:
        approved:
          type: boolean
        comment:
          type: string
          default: ""

  - alias: finalize
    type: agent_call
    executor: mock
    prompt: "用户决定：{{confirm.approved}}，评论：{{confirm.comment}}"
    output: summary
```

**操作步骤**：

```bash
# 1. 写入 workflow 文件
cat > /tmp/aw-smoke/wait-user-test.yaml <<'EOF'
name: wait-user-test
version: "1.0"
description: 测试 wait_user 节点的暂停与恢复
nodes:
  - alias: generate
    type: agent_call
    executor: mock
    prompt: "生成一段文字"
    output: content
  - alias: confirm
    type: wait_user
    description: 让用户确认是否采纳
    message: "生成的内容是：{{content}}\n你是否采纳？"
    schema:
      type: object
      required: [approved]
      properties:
        approved:
          type: boolean
        comment:
          type: string
          default: ""
  - alias: finalize
    type: agent_call
    executor: mock
    prompt: "用户决定：{{confirm.approved}}，评论：{{confirm.comment}}"
    output: summary
EOF

# 2. 启动（mock 会先跑 generate，然后停在 confirm 等你）
AGENT_WORKFLOW_ENABLE_MOCK=1 \
AGENT_WORKFLOW_MOCK_GENERATE="这是AI生成的文章草稿" \
AGENT_WORKFLOW_MOCK_FINALIZE="最终确认完毕" \
python3 skills/agent-workflow/tool.py start '{"workflow":"/tmp/aw-smoke/wait-user-test.yaml","caller":"manual-test"}'
```

**预期**：返回 `action: "wait_user"`，`payload.message` 里包含 generate 的输出。

```bash
# 3. 用 resume 模拟用户确认（把返回的 run_id 填进去）
AGENT_WORKFLOW_ENABLE_MOCK=1 \
AGENT_WORKFLOW_MOCK_FINALIZE="最终确认完毕" \
python3 skills/agent-workflow/tool.py resume '{"run_id":"<上面返回的run_id>","input":{"approved":true,"comment":"写得不错"}}'
```

**预期**：返回 `action: "completed"`，`vars.summary` = `"最终确认完毕"`。

**验证要点**：
- start 时 workflow 能正确暂停在 wait_user
- resume 能传入结构化数据并继续后续节点
- `{{confirm.approved}}` 和 `{{confirm.comment}}` 被正确渲染进 finalize 的 prompt

## 测试 7：Schema 校验错误 — 传了不合法的 user_input

**场景故事**：紧接测试 6，如果你 resume 时忘了必填字段，系统应该友好报错。

**操作**（使用测试 6 的同一个 run，如果已经完成就重新 start 一个）：

```bash
# 重新启动一个新的
AGENT_WORKFLOW_ENABLE_MOCK=1 \
AGENT_WORKFLOW_MOCK_GENERATE="测试内容" \
AGENT_WORKFLOW_MOCK_FINALIZE="done" \
python3 skills/agent-workflow/tool.py start '{"workflow":"/tmp/aw-smoke/wait-user-test.yaml","caller":"manual-test"}'

# 用错误输入 resume（缺少 required 的 approved 字段）
python3 skills/agent-workflow/tool.py resume '{"run_id":"<run_id>","input":{"comment":"只填了评论"}}'
```

**预期**：返回 error，code 为 `SCHEMA_VIOLATION`，`retryable: true`，message 提示 `approved` 字段缺失。

**验证要点**：
- 错误是 retryable，意味着你可以修正输入后再 resume
- workflow 没有进入终态（仍然是 waiting_user）

```bash
# 修正后再次 resume
AGENT_WORKFLOW_ENABLE_MOCK=1 \
AGENT_WORKFLOW_MOCK_FINALIZE="done" \
python3 skills/agent-workflow/tool.py resume '{"run_id":"<run_id>","input":{"approved":false,"comment":"需要修改"}}'
```

**预期**：这次成功，返回 completed。

## 测试 8：Loop 循环 — 反复迭代直到通过

**场景故事**：你启动一个"起草→审阅→改稿"的循环 workflow，前两次审阅拒绝（approved=false），第三次通过（approved=true），循环结束。

**workflow 文件** `/tmp/aw-smoke/loop-test.yaml`：

```yaml
name: loop-test
version: "1.0"
description: 测试 loop 节点 — 循环到用户满意

vars:
  topic: "AI安全"

nodes:
  - alias: first_draft
    type: agent_call
    executor: mock
    prompt: "写一段关于{{topic}}的初稿"
    output: draft

  - alias: refine_loop
    type: loop
    condition: "{{review.approved}} == false"
    max_iterations: 5
    body:
      - alias: review
        type: wait_user
        message: "当前稿件：{{draft}}\n是否通过？"
        schema:
          type: object
          required: [approved]
          properties:
            approved:
              type: boolean
            feedback:
              type: string
              default: ""
      - alias: refine
        type: agent_call
        executor: mock
        prompt: "根据反馈修改：{{review.feedback}}"
        output: draft
```

**操作步骤**：

```bash
cat > /tmp/aw-smoke/loop-test.yaml <<'EOF'
name: loop-test
version: "1.0"
description: 测试 loop 节点 — 循环到用户满意
vars:
  topic: "AI安全"
nodes:
  - alias: first_draft
    type: agent_call
    executor: mock
    prompt: "写一段关于{{topic}}的初稿"
    output: draft
  - alias: refine_loop
    type: loop
    condition: "{{review.approved}} == false"
    max_iterations: 5
    body:
      - alias: review
        type: wait_user
        message: "当前稿件：{{draft}}\n是否通过？"
        schema:
          type: object
          required: [approved]
          properties:
            approved:
              type: boolean
            feedback:
              type: string
              default: ""
      - alias: refine
        type: agent_call
        executor: mock
        prompt: "根据反馈修改：{{review.feedback}}"
        output: draft
EOF

# 启动
AGENT_WORKFLOW_ENABLE_MOCK=1 \
AGENT_WORKFLOW_MOCK_FIRST_DRAFT="AI安全初稿v1" \
AGENT_WORKFLOW_MOCK_REFINE="AI安全修订版" \
python3 skills/agent-workflow/tool.py start '{"workflow":"/tmp/aw-smoke/loop-test.yaml","caller":"manual-test"}'
```

**预期**：mock 执行 first_draft 后进入 loop body 的 wait_user，返回 `action: "wait_user"`。

```bash
# 第一轮：拒绝
AGENT_WORKFLOW_ENABLE_MOCK=1 \
AGENT_WORKFLOW_MOCK_REFINE="AI安全修订v2" \
python3 skills/agent-workflow/tool.py resume '{"run_id":"<run_id>","input":{"approved":false,"feedback":"需要更详细"}}'
```

**预期**：refine 节点用 mock 输出后，循环检查 condition（approved==false → 继续循环），再次 wait_user。

```bash
# 第二轮：拒绝
AGENT_WORKFLOW_ENABLE_MOCK=1 \
AGENT_WORKFLOW_MOCK_REFINE="AI安全修订v3" \
python3 skills/agent-workflow/tool.py resume '{"run_id":"<run_id>","input":{"approved":false,"feedback":"还差一点"}}'
```

**预期**：同上，再次 wait_user。

```bash
# 第三轮：通过
python3 skills/agent-workflow/tool.py resume '{"run_id":"<run_id>","input":{"approved":true,"feedback":""}}'
```

**预期**：condition 判定 `approved == false` 为 false → 循环终止 → workflow completed。`vars.draft` 应该是最后一次 refine 的输出。

**验证要点**：
- loop 的 do-while 语义（首次 condition 中变量未定义时仍进入 body）
- 条件判定正确：false 继续、true 终止
- 每轮 refine 的 output 覆盖了 `vars.draft`

## 测试 9：Sleep 节点 — 原子等待

**场景故事**：在两个推理节点之间插入一个 3 秒的等待（模拟限流场景）。

**workflow 文件** `/tmp/aw-smoke/sleep-test.yaml`：

```yaml
name: sleep-test
version: "1.0"
description: 测试 sleep 节点的阻塞行为

nodes:
  - alias: step1
    type: agent_call
    executor: mock
    prompt: "第一步"
    output: result1

  - alias: pause
    type: sleep
    seconds: 3

  - alias: step2
    type: agent_call
    executor: mock
    prompt: "第二步，上一步结果：{{result1}}"
    output: result2
```

**操作步骤**：

```bash
cat > /tmp/aw-smoke/sleep-test.yaml <<'EOF'
name: sleep-test
version: "1.0"
description: 测试 sleep 节点的阻塞行为
nodes:
  - alias: step1
    type: agent_call
    executor: mock
    prompt: "第一步"
    output: result1
  - alias: pause
    type: sleep
    seconds: 3
  - alias: step2
    type: agent_call
    executor: mock
    prompt: "第二步，上一步结果：{{result1}}"
    output: result2
EOF

# 启动（注意计时 — 应该比无 sleep 慢约 3 秒）
time AGENT_WORKFLOW_ENABLE_MOCK=1 \
AGENT_WORKFLOW_MOCK_STEP1="done-step1" \
AGENT_WORKFLOW_MOCK_STEP2="done-step2" \
python3 skills/agent-workflow/tool.py start '{"workflow":"/tmp/aw-smoke/sleep-test.yaml","caller":"manual-test"}'
```

**预期**：
- 返回 `completed`，`vars.result1="done-step1"`, `vars.result2="done-step2"`
- 整个命令耗时 ≥ 3 秒（因为中间 sleep 了 3 秒）
- events 中能看到 `node_start` / `node_end` 对应 pause 节点，duration ≈ 3000ms

## 测试 13：Validate 校验失败 — 各种 YAML 错误

**场景故事**：你写了几个有问题的 YAML，让 validate 逐一报错，验证 4 级校验的诊断能力。

### 13a：引用了不存在的变量（L3 数据流错误）

```bash
cat > /tmp/aw-smoke/bad-var.yaml <<'EOF'
name: bad-var
version: "1.0"
description: 引用了未定义变量
nodes:
  - alias: step1
    type: agent_call
    executor: mock
    prompt: "处理 {{nonexistent_var}}"
    output: result
EOF

python3 skills/agent-workflow/tool.py validate '{"workflow":"/tmp/aw-smoke/bad-var.yaml"}'
```

**预期**：violations 里报 `VAR_NOT_IN_SCOPE`，指出 `nonexistent_var` 在 step1 的 prompt 中引用但未定义。

### 13b：alias 重复（L3 唯一性错误）

```bash
cat > /tmp/aw-smoke/dup-alias.yaml <<'EOF'
name: dup-alias
version: "1.0"
description: 两个节点 alias 相同
nodes:
  - alias: step
    type: agent_call
    executor: mock
    prompt: "第一步"
    output: r1
  - alias: step
    type: agent_call
    executor: mock
    prompt: "第二步 {{r1}}"
    output: r2
EOF

python3 skills/agent-workflow/tool.py validate '{"workflow":"/tmp/aw-smoke/dup-alias.yaml"}'
```

**预期**：violations 报 alias `step` 重复。

### 13c：executor 不在 PATH（L4 环境检查）

```bash
cat > /tmp/aw-smoke/bad-executor.yaml <<'EOF'
name: bad-executor
version: "1.0"
description: 用了一个不存在的 executor
nodes:
  - alias: step1
    type: agent_call
    executor: nonexistent_llm
    prompt: "hello"
    output: result
EOF

python3 skills/agent-workflow/tool.py validate '{"workflow":"/tmp/aw-smoke/bad-executor.yaml"}'
```

**预期**：violations 报 L4 级别错误，`nonexistent_llm` 找不到。

## 测试 14：Agent 上下文配置（role + skills）

**场景故事**：验证 agent_call 节点的 `agent` 字段能正确传递 role 和 skills 到 payload 中。

name: agent-context-test

- `payload.prompt` = "设计一个缓存方案"
- `payload.agent.role` 包含 architect 角色描述
- `payload.agent.skills` = `["knowledge-compiler", "swagger-api-reader"]`

```bash
# advance 结束它
python3 skills/agent-workflow/tool.py advance '{"run_id":"<run_id>","result":{"output":"使用 Redis + 本地 LRU 二级缓存"}}'
```

**预期**：completed，`vars.design` = 你 advance 传的内容。

## 测试 15：vars 初始化 + ENV 环境变量注入

**场景故事**：验证 start 时可以传 vars，以及 `$ENV:` 占位符能从系统环境变量展开。

```bash
cat > /tmp/aw-smoke/vars-test.yaml <<'EOF'
name: vars-test
version: "1.0"
description: 测试 vars 初始化和 ENV 展开

vars:
  project_name: ""
  api_key: "$ENV:TEST_API_KEY"
  _secrets: ["api_key"]

nodes:
  - alias: use_vars
    type: agent_call
    executor: mock
    prompt: "项目{{project_name}}的key长度={{api_key}}"
    output: result
EOF

# 设置环境变量后启动
TEST_API_KEY="sk-fake-12345" \
AGENT_WORKFLOW_ENABLE_MOCK=1 \
AGENT_WORKFLOW_MOCK_USE_VARS="processed" \
python3 skills/agent-workflow/tool.py start '{
  "workflow":"/tmp/aw-smoke/vars-test.yaml",
  "caller":"manual-test",
  "vars":{"project_name":"skillix-hub"}
}'
```

**预期**：
- completed，vars 中 `project_name = "skillix-hub"`
- `api_key` 被 `$ENV:TEST_API_KEY` 展开为 `"sk-fake-12345"`
- 但在 events/audit 中，api_key 的值应被替换为 `***REDACTED***`（因为列入了 _secrets）

验证 secrets 脱敏：

```bash
# 查看 events，确认 api_key 被脱敏
python3 skills/agent-workflow/tool.py status '{"run_id":"<run_id>","include_events":true}'
```
