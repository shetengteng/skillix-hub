> 工具：OpenCode + `run-opencode.sh` wrapper

## 行为差异分析

### 1. Skill 发现能力

| 维度 | Sonnet | Haiku |
|---|---|---|
| 使用 `skill` tool | 100%（每次首选） | 60%（不稳定，有时手动读目录） |
| 加载 SKILL.md 方式 | skill tool 自动加载 | 有时手动 `read SKILL.md`（分段读取） |
| 首次发现成功率 | 高 | 中（Test 1 完全跳过 skill tool） |

### 2. CLI 调用准确率

| 维度 | Sonnet | Haiku |
|---|---|---|
| 首次调用成功率 | ~90% | ~50% |
| 常见错误 | 偶尔 JSON 引号问题 | `UNKNOWN_ACTION (run)`、`PARAMS_INVALID`、JSON 转义 |
| 自修复能力 | 快速（1 次重试） | 较慢（需读文档确认后重试，2-3 次） |
| 命令格式偏好 | `agent-workflow <action> '<JSON>'` | 有时用 `python3 tool.py`，有时写临时文件传 JSON |

### 3. 主循环执行

| 维度 | Sonnet | Haiku |
|---|---|---|
| action 路由 | 严格按 SKILL.md 4 种 action 处理 | 基本正确，但偶尔漏掉步骤 |
| `resume` 参数名 | `input`（修复后） | `input`（正确） |
| advance 处理 | 正确 | 正确 |
| 错误恢复策略 | 直接重试 | 先读文档再重试（更稳但更慢） |

### 4. 性能

| 测试 | Sonnet 耗时 | Haiku 耗时 | 差异 |
|---|---|---|---|
| Test 1 | 8s | 29s | Haiku 3.6x 慢（手动读文件） |
| Test 2 | 30s | 23s | Haiku 更快（简单查询） |
| Test 6 | 57s | 92s+ | Haiku 1.6x 慢（多次重试） |
| Test 9 | 67s | 43s | Haiku 更快（含 3s sleep） |
| Test 13 | 57s | 25s | Haiku 2.3x 快（并行执行） |

**结论**：Haiku 在简单查询场景（Test 2, 13）与 Sonnet 持平甚至更快，但在复杂多步操作（Test 1, 6）明显更慢且错误率更高。

## 关键发现

### Haiku 的优势
1. **简单任务执行快** — Test 2 (23s) 和 Test 13 (25s) 比 Sonnet 更快
2. **并行调用能力强** — Test 13 中三个 validate 命令被并行执行
3. **最终结果正确** — 所有测试都通过了功能验证

### Haiku 的劣势
1. **Skill Tool 使用不稳定** — 有时跳过 skill tool 直接手动读文件，导致首次加载慢
2. **CLI 命令格式掌握弱** — 首次调用经常出现 `UNKNOWN_ACTION` 或 `PARAMS_INVALID`
3. **JSON Shell 转义问题** — 在 shell 中构造 JSON 时经常出错，需要用临时文件绕过
4. **多步任务效率低** — wait_user 完整流程需要多次重试，超时风险高

### 改进建议

| 问题 | SKILL.md 可做的优化 |
|---|---|
| Haiku 不认识 `start` action | 在 CLI 速查表中加粗 action 名，或在 frontmatter 添加 `actions: [start, advance, resume, ...]` |
| JSON 构造出错 | 在 SKILL.md 中提供完整 shell 命令示例（含引号转义），或建议用临时文件 |
| Skill tool 使用不稳定 | 不在 SKILL.md 控制范围内，属于 OpenCode/Haiku 的 skill discovery 行为 |
| 多步操作超时 | 建议将复杂多步操作拆成独立指令，而不是一个长 prompt |

---

## 模型推荐

| 场景 | 推荐模型 | 原因 |
|---|---|---|
| 简单查询（executors, templates, validate） | Haiku | 更快更便宜 |
| 完整 workflow 执行（start→advance→resume） | Sonnet | 首次成功率高，不需要多次重试 |
| 调试/排错（status, events 分析） | Sonnet | 理解复杂 JSON 输出更准确 |
| CI/自动化批量验证 | Haiku + 重试机制 | 成本低，配合重试可达到可靠性 |
