# 长期记忆生成逻辑验证 Demo

## 验证目标

1. **MEMORY.md 自动生成**：事实保存 → distill 提炼 → MEMORY.md 写入
2. **USER_MEMORY.md 行为**：不自动生成，手动创建后被加载
3. **三阶段质量提升**：入口防污染 → 提示词引导 → LLM 精炼

---

## 测试 1：事实保存与 MEMORY.md 生成

### 1.1 保存事实

```bash
# 保存几条不同类型的事实
python3 ~/.cursor/skills/memory/scripts/service/memory/save_fact.py \
  --content "项目使用 PostgreSQL 作为主数据库" \
  --type W --entities "PostgreSQL,数据库" --confidence 0.95 --session "demo-test-1"

python3 ~/.cursor/skills/memory/scripts/service/memory/save_fact.py \
  --content "用户偏好使用 TypeScript 而非 JavaScript" \
  --type O --entities "TypeScript,偏好" --confidence 0.9 --session "demo-test-1"

python3 ~/.cursor/skills/memory/scripts/service/memory/save_fact.py \
  --content "skillix-hub 是一个 AI Agent Skill 管理平台" \
  --type B --entities "skillix-hub,项目" --confidence 1.0 --session "demo-test-1"
```

### 1.2 验证事实已保存

```bash
# 查看今日 daily 文件
cat .cursor/skills/memory-data/daily/$(date +%Y-%m-%d).jsonl | python3 -m json.tool
```

预期：每条事实都有 `id`、`content`、`memory_type`、`confidence`、`timestamp`。

### 1.3 触发 distill（规则提炼）

```bash
python3 ~/.cursor/skills/memory/scripts/service/memory/distill_to_memory.py \
  --project-path $(pwd)
```

### 1.4 验证 MEMORY.md 更新

```bash
cat .cursor/skills/memory-data/MEMORY.md
```

预期：事实按类型分类写入对应 section（用户偏好 / 项目背景 / 重要决策）。

## 测试 2：Stage 0 入口防污染

### 2.1 命令注入过滤

```bash
# 包含 save_fact.py 命令的脏数据
python3 ~/.cursor/skills/memory/scripts/service/memory/save_fact.py \
  --content "python3 save_fact.py --content '用户偏好中文' --type O 用户偏好中文回答" \
  --type O --entities "偏好" --session "demo-test-2"
```

预期：保存后 content 中的 `python3 save_fact.py ...` 片段被清除，只保留有效内容。

### 2.2 超长内容截断

```bash
python3 ~/.cursor/skills/memory/scripts/service/memory/save_fact.py \
  --content "$(python3 -c "print('这是一段非常长的内容' * 100)")" \
  --type W --entities "测试" --session "demo-test-2"
```

预期：content 被截断到 800 字符以内。

### 2.3 memory_type 自动推断

```bash
# 声明 type W，但内容是用户偏好
python3 ~/.cursor/skills/memory/scripts/service/memory/save_fact.py \
  --content "用户偏好使用 Vim 编辑器" \
  --type W --entities "偏好" --session "demo-test-2"
```

预期：保存后 `memory_type` 被自动推断为 `O`（而非声明的 `W`）。

验证方式：

```bash
# 查看最后一条记录的 memory_type
tail -1 .cursor/skills/memory-data/daily/$(date +%Y-%m-%d).jsonl | python3 -m json.tool | grep memory_type
```

## 测试 3：USER_MEMORY.md 行为

### 3.1 确认不自动生成

```bash
ls -la .cursor/skills/memory-data/USER_MEMORY.md 2>&1
```

预期：文件不存在（`No such file or directory`）。

### 3.2 手动创建后验证加载

```bash
# 手动创建
cat > .cursor/skills/memory-data/USER_MEMORY.md << 'EOF'
# 用户自定义记忆

## 个人备忘
- 每周五下午做代码 review
- 部署前必须跑完整测试套件

## 项目约定
- API 响应统一使用 camelCase
- 错误码前缀：AUTH=1xxx, BIZ=2xxx, SYS=9xxx
EOF

echo "USER_MEMORY.md 已创建"
```

### 3.3 验证 load_memory 加载

```bash
python3 ~/.cursor/skills/memory/scripts/service/hooks/load_memory.py --project-path $(pwd)
```

预期输出中包含：
- `## 核心记忆`（来自 MEMORY.md）
- `## 用户自定义记忆`（来自 USER_MEMORY.md）

### 3.4 验证 distill 不修改 USER_MEMORY.md

```bash
# 记录修改时间
stat -f "%Sm" .cursor/skills/memory-data/USER_MEMORY.md

# 执行 distill
python3 ~/.cursor/skills/memory/scripts/service/memory/distill_to_memory.py \
  --project-path $(pwd)

# 再次检查修改时间
stat -f "%Sm" .cursor/skills/memory-data/USER_MEMORY.md
```

预期：两次时间戳相同，USER_MEMORY.md 未被修改。
