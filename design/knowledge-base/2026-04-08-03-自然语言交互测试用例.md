# Knowledge Base Skill 自然语言交互测试用例

> 日期：2026-04-08
> 目的：验证 KB Skill 在真实对话中的可用性，用户通过自然语言与 AI Agent 交互，Agent 自动识别意图并调用 KB 命令

## 前提条件

- 已在项目中安装了 knowledge-base skill（`python3 skills/knowledge-base/main.py install`）
- 依赖已安装（`pip3 install typer pydantic pyyaml`）
- SKILL.md 已被 Agent 识别（在 `.cursor/skills/` 或 `skills/` 路径下）

## 测试 1：首次安装与初始化

```
用户: 帮我初始化知识库
预期: Agent 执行 python3 skills/knowledge-base/main.py install
      输出 "✅ Knowledge Base Skill 安装完成" 及数据目录路径
验证: knowledge-base-data/ 目录及子目录（raw/, wiki/, compile/）已创建
```

## 测试 2：导入项目设计文档

```
用户: 把项目的设计文档都导入知识库
预期: Agent 执行 python3 skills/knowledge-base/main.py import-project
      输出 "✅ 项目导入完成: N 个新条目"
验证: python3 skills/knowledge-base/main.py list 输出的条目数 > 0
```

## 测试 3：添加单个代码文件

```
用户: 把 skills/knowledge-base/src/indexer.py 加到知识库，标签是 indexer, core
预期: Agent 执行 python3 skills/knowledge-base/main.py add skills/knowledge-base/src/indexer.py --tags "indexer,core"
      输出 "✅ 已添加" + 类型显示为 code
验证: python3 skills/knowledge-base/main.py list --type code 显示该条目
```

## 测试 4：查看索引列表

```
用户: 看看知识库里有什么
预期: Agent 执行 python3 skills/knowledge-base/main.py list
      输出按分类分组的索引列表，含 ⏳/✅ 状态标记
验证: 输出格式包含 "📂 分类名 (N 个)" 和条目 ID
```

## 测试 5：查看待编译条目

```
用户: 有哪些资料还没编译
预期: Agent 执行 python3 skills/knowledge-base/main.py list --pending
      输出只含 compiled=false 的条目
验证: 所有条目前面都是 ⏳ 标记
```

## 测试 6：预览编译清单

```
用户: 看一下待编译的清单
预期: Agent 执行 python3 skills/knowledge-base/main.py compile --dry-run
      输出待编译条目列表及数量
验证: 输出包含 "共 N 个条目待编译"
```

## 测试 7：执行编译

```
用户: 编译知识库
预期:
  1. Agent 执行 python3 skills/knowledge-base/main.py compile
  2. Agent 阅读输出的 prompt，分析源材料内容
  3. Agent 在 wiki/concepts/ 中创建概念条目 .md 文件（含 frontmatter）
  4. Agent 执行 python3 skills/knowledge-base/main.py compile --finalize
  5. 输出 "编译后处理完成" + 概念数/图谱/地图路径
验证:
  - wiki/concepts/ 目录下有 .md 文件
  - wiki/index.md 知识地图已生成
  - wiki/graph.json 知识图谱已生成
  - wiki/backlinks.json 反向链接已生成
```

## 测试 8：浏览知识库（渐进式披露）

```
# 第一层：知识地图
用户: 浏览一下知识库
预期: Agent 执行 python3 skills/knowledge-base/main.py browse
      输出知识地图（按分类列出概念列表）

# 第二层：分类索引
用户: 看看 memory 分类
预期: Agent 执行 python3 skills/knowledge-base/main.py browse memory
      输出 memory 分类下的概念列表

# 第三层：概念条目
用户: 看看四层架构的详细内容
预期: Agent 执行 python3 skills/knowledge-base/main.py read memory-four-layer-architecture
      输出概念条目全文（含核心要点、关联概念、来源资料）

# 第四层：来源资料
用户: 看一下这个概念的来源资料详情
预期: Agent 执行 python3 skills/knowledge-base/main.py source kb-20260408-024455-025
      输出资料路径、类型、标签、内容预览
```

## 测试 9：搜索

```
用户: 搜一下跟 Hook 相关的内容
预期: Agent 执行 python3 skills/knowledge-base/main.py search "Hook"
      输出包含索引条目（📄 标记）和概念条目（💡 标记）的搜索结果

用户: 找找之前关于 pywebview 的设计
预期: Agent 识别 "之前的设计" 触发 KB skill
      执行 search "pywebview"
      输出相关条目
验证: 搜索结果包含 agent-interact 分类的条目
```

## 测试 10：知识图谱

```
用户: 看一下知识图谱
预期: Agent 执行 python3 skills/knowledge-base/main.py graph --format mermaid
      输出 Mermaid 格式的图谱（graph LR 开头）

用户: 以 memory-four-layer-architecture 为中心看关联
预期: Agent 执行 python3 skills/knowledge-base/main.py graph --center memory-four-layer-architecture --depth 2
      输出以该概念为中心、深度 2 的子图
验证: 子图只包含与 memory-four-layer-architecture 距离 ≤ 2 的节点
```

## 测试 11：状态检查

```
用户: 知识库状态怎么样
预期: Agent 执行 python3 skills/knowledge-base/main.py status
      输出索引统计（总条目/已编译/待编译）+ Wiki 统计（概念数/图谱状态）
验证: 数字正确，与 list 命令的输出一致

用户: 检查一下有没有失效的路径
预期: Agent 执行 python3 skills/knowledge-base/main.py check
      输出路径有效性分类统计（✅ 有效 / ❌ 失效 / 🔄 变更）
```

## 测试 12：概念管理

```
用户: 列一下所有概念
预期: Agent 执行 python3 skills/knowledge-base/main.py concept list
      输出概念 ID 列表

用户: 把 incremental-compilation 这个概念重命名为"增量编译策略"
预期: Agent 执行 python3 skills/knowledge-base/main.py concept rename incremental-compilation "增量编译策略"
      输出 "✅ 已重命名"
验证: python3 skills/knowledge-base/main.py read incremental-compilation 显示新标题
```
