# 质量等级定义

## 等级划分

| 等级 | 标准 | 说明 |
|-----|------|------|
| ⭐⭐⭐ 优秀 | 所有检查项通过 | 可直接使用 |
| ⭐⭐ 良好 | 无"需修改"项，有"可优化"项 | 建议优化后使用 |
| ⭐ 合格 | 有"需修改"项，但核心功能完整 | 需要修改后使用 |
| ❌ 不合格 | 缺少必需章节或核心功能缺失 | 需要重大修改 |

---

## 检查项分类

### 必须通过（否则不合格）

**结构类**：
- 必需章节齐全
- 章节顺序合理

**内容类**：
- 核心职责明确
- 工作流程完整
- 前端交互格式正确（如适用）

### 建议修改

**内容类**：
- 定义与执行分离
- 场景位置合理
- 输出格式位置合理

**风格类**：
- 避免表格
- 示例使用占位符

### 可选优化

**风格类**：
- 术语一致性
- 语言自然流畅
- 解释充分

## 评分示例

### 示例 1：优秀

```
## Prompt 质量检测报告

**Agent 名称**：Classification_Classifier_V2
**Agent 类型**：Standard

### 结果
- ✅ 结构完整（5/5）
- ✅ 内容规范（6/6）
- ✅ 风格一致（4/4）

### 总结
- **等级**：⭐⭐⭐ 优秀
- **建议**：可直接使用
```

### 示例 2：良好

```
## Prompt 质量检测报告

**Agent 名称**：Metadata_Companion
**Agent 类型**：Standard

### 结果
- ✅ 结构完整（5/5）
- ✅ 内容规范（6/6）
- ⚠️ 风格检查（3/4）
  - 发现 1 处表格 → 建议改为列表

### 总结
- **等级**：⭐⭐ 良好
- **建议**：修改表格后可使用
```

### 示例 3：合格

```
## Prompt 质量检测报告

**Agent 名称**：New_Agent
**Agent 类型**：Standard

### 结果
- ✅ 结构完整（5/5）
- ❌ 内容规范（4/6）
  - 执行区重复定义
  - 缺少必需字段 logicalTableId
- ⚠️ 风格检查（3/4）

### 总结
- **等级**：⭐ 合格
- **建议**：需要修改内容问题
```

### 示例 4：不合格

```
## Prompt 质量检测报告

**Agent 名称**：Draft_Agent
**Agent 类型**：Standard

### 结果
- ❌ 结构完整（2/5）
  - 缺少"核心任务"章节
  - 缺少"工作流程"章节
  - 缺少"输出格式"章节

### 总结
- **等级**：❌ 不合格
- **建议**：需要补充必需章节
```

---

## 持续改进

质量检测不是一次性的，建议：

1. **创建后检测**：新 Prompt 创建后立即检测
2. **修改后复查**：修改 Prompt 后重新检测
3. **定期审查**：定期审查现有 Prompt 的质量
