# 常见问题和修复方法

## 结构问题

### 1. 缺少必需章节

**问题**：Prompt 缺少关键章节，如"核心任务"、"工作流程"。

**修复**：按模板补充章节。

**Standard Agent 模板**：
```markdown
# 角色

[Agent 身份和定位]

## 核心任务

[主要职责描述]

**关键原则**：
- ...

## 工作流程

### 步骤 1：...
### 步骤 2：...

## 输出格式

（如适用）
```

---

### 2. 章节顺序混乱

**问题**：章节顺序不符合阅读习惯，如输出格式在工作流程之前。

**修复**：调整为标准顺序。

**推荐顺序**：
1. 角色/背景
2. 核心任务/职责
3. 输入格式（如有）
4. 工作流程
5. 输出格式
6. 核心原则/约束

---

## 内容问题

### 3. 执行区重复定义

**问题**：执行区解释"是什么"，重复了定义区的内容。

**示例（错误）**：
```markdown
### 步骤 3：执行查询

**如果是「发现」场景**（用户想查找数据资产，典型问题如"查找支付相关的表"）：
- 使用 multi_match 搜索
```

**修复**：删除括号内的解释，只保留动作。

**示例（正确）**：
```markdown
### 步骤 3：执行查询

**如果是「发现」场景**：
- 使用 multi_match 搜索
```

---

### 4. 场景位置不当

**问题**：场景定义位置与引用频率不匹配。

**判断方法**：
- 多步骤引用场景详情 → 放在工作流程之前
- 单步骤引用场景详情 → 放在步骤中
- 只需名称引用 → 放在独立章节

---

### 5. 输出格式缺少必需字段

**问题**：前端交互格式缺少 `id`、`logicalTableId` 等必需字段。

**修复**：检查前端解析逻辑，补充必需字段。

**必需字段**：
```json
{
  "id": "字段ID（必需）",
  "logicalTableId": "表ID（必需）",
  ...
}
```

---

## 风格问题

### 6. 使用表格

**问题**：Prompt 中使用 Markdown 表格，对模型解析不友好。

**修复**：改为列表或分段描述。

**示例（错误）**：
```markdown
| 场景 | 处理方式 |
|-----|---------|
| 发现 | multi_match |
| 分析 | terms 聚合 |
```

**示例（正确）**：
```markdown
- **发现**：使用 multi_match 搜索
- **分析**：使用 terms 聚合
```

---

### 7. 示例使用具体值

**问题**：示例使用具体值，导致 AI 照搬。

**示例（错误）**：
```json
{
  "id": "field_123",
  "fieldName": "user_name",
  "termId": "TERM_456"
}
```

**修复**：使用 high-level 占位符。

**示例（正确）**：
```json
{
  "id": "字段ID",
  "fieldName": "字段名称",
  "termId": "匹配的术语ID"
}
```

---

### 8. 语言过于机械

**问题**：语言像命令脚本，缺少解释和上下文。

**示例（错误）**：
```markdown
步骤1：调用工具
步骤2：解析数据
步骤3：输出结果
```

**修复**：添加解释和原因。

**示例（正确）**：
```markdown
### 步骤 1：获取数据

调用 MMS_GetUnclassifiedFields 获取未分类字段。注意检查返回数据是否完整，空数据需要提示用户。

### 步骤 2：分析字段

逐个分析每个字段，根据字段名和数据类型匹配术语。优先完全匹配，其次语义匹配。
```

---

### 9. 术语不一致

**问题**：同一概念使用不同术语，如"场景"和"意图"混用。

**修复**：选择一个术语，全文统一。

**常见术语对**：
- 场景 / 意图 → 统一用"场景"
- 步骤 / 阶段 → 根据粒度选择
- 输出 / 返回 → 统一用"输出"
