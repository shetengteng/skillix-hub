# Prompt 示例值编写规范

## 核心原则

**示例展示模式，不展示具体数据。**

- 结构化示例（JSON）：用 high-level 占位符展示结构
- 自然语言示例（情境模板）：用参考样例展示语气和内容范围

---

## Facet 定位

示例值是**跨 Facet 类型的通用写作技能**，不同类型的示例服务不同的 Facet：

**结构化示例（JSON 占位符）**：
- 服务 **规范型** facet — 输出格式示例（`async_cui_callback`、`async_cui_suggest`）
- 服务 **知识型** facet — 输入格式示例（数据结构、Agent 接口参数）
- 深度指南 → `@guide-output`

**自然语言示例（情境模板）**：
- 服务 **规范型** facet — NL 输出的语气和内容范围参考
- 服务 **边界型** facet — 边界情况（无结果、数据为空）的处理参考
- 深度指南 → `@guide-output`（Part：自然语言输出设计）

**识别该用哪种示例**：
- 模型需要知道"输出结构长什么样" → 结构化示例
- 模型需要知道"该用什么语气、说什么内容" → 自然语言示例

---

## 1. 结构化示例：High-level 占位符

### 为什么用占位符？

**示例更清晰**：
- 关注结构，不关注数据
- 展示"格式是什么"，而非"数据是什么"
- 避免让模型误以为示例值就是实际值

**更灵活**：
- 适用于各种场景
- 不绑定特定业务数据
- 占位符让模型理解"这是变量"

如果用具体值：
- ❌ 示例变成"真实数据"，让模型困惑
- ❌ 容易被误解为"只能用这些值"
- ❌ 具体值转移注意力，忽略了结构本身

### 占位符类型

**ID 类型**：
- 用 `"字段ID"`、`"表ID"`、`"术语ID"`
- 不用 `"1991266546175094787"`

**数字类型**：
- 用 `X`、`Y`、`count`、`total`
- 不用 `15`、`20`、`3`

**文本类型**：
- 用 `"字段名"`、`"术语名称"`、`"结果值"`
- 不用具体值（除非是字段名示例）

**数组类型**：
- 只需 1-2 个元素展示结构
- 不要堆砌多个相似元素

### 流式输出：边处理边输出

**处理完一个对象，就输出这个对象，不等全部处理完。**

✅ **推荐（边处理边输出）**：

```markdown
正在分析字段 1...

```json async_cui_callback
{"id": "字段ID", "logicalTableId": "表ID", "result": "分类结果"}
```（反引号闭合）

继续分析字段 2...

```json async_cui_callback
{"id": "字段ID", "logicalTableId": "表ID", "result": "分类结果"}
```（反引号闭合）
```

**关键**：
- ✅ 单个对象立即输出
- ✅ 用户能看到实时进度
- ❌ 不要等全部完成再批量输出

## 2. 自然语言示例：情境模板

### 为什么需要情境模板？

模型在"没问题"或"特殊情况"时往往不知道该说什么，要么输出过于正式，要么生硬地套用"有问题"时的格式。情境模板给模型一个参考样例，展示在不同情况下应该用什么语气、说什么内容。

### 写法

列出每种情况，给出自然语言的参考样例：

```markdown
**有问题时**：
有 [X] 个字段填写的内容有点问题：[自然地描述问题]

**无问题时**：
都挺好的，描述和示例值都填写规范。

**大部分没填时**：
看了一圈，大部分字段的描述和示例值都还没填，暂时没什么质量问题可检测。
```

### 关键原则

- 模板是**参考样例**，不是死板格式。模型会根据实际情况灵活调整措辞
- 每种情况只需要一两句话，展示语气和内容范围即可
- 覆盖常见情况，特别是"没问题"和"边界情况"——这些是模型最容易出错的地方

### 与结构化示例的统一原则

结构化示例用占位符（如 `"字段ID"`）避免模型照搬具体值。自然语言示例同理：模板展示的是语气和内容范围的**模式**，不是让模型逐字复制。

---

## 检查清单

### 结构化示例（JSON）
- [ ] 是否用了具体数字/ID？（应该用 `"字段ID"`、`X`）
- [ ] 字符串值是否用占位符？（应该用 `"字段名"` 而非 `"user_email"`）
- [ ] JSON 示例是否语法正确？（括号、引号、逗号）
- [ ] 是否体现边处理边输出？（单个对象，不是数组）
- [ ] 数组是否简洁（1-2 个元素）？

### 自然语言示例（情境模板）
- [ ] 是否覆盖了"有结果"的情况？
- [ ] 是否覆盖了"没问题/没结果"的情况？
- [ ] 是否覆盖了"边界情况"？（数据为空、大部分缺失等）
- [ ] 模板是否展示了期望的语气？

### 整体
- [ ] 示例是否保持 high-level 抽象？
- [ ] 具体例子是否在文字中说明？

---

## 总结

**核心**：示例展示模式，不展示具体数据。

**两种示例**：
1. 结构化示例 — 用 high-level 占位符展示结构，边处理边输出
2. 自然语言示例 — 用情境模板展示语气和内容范围，覆盖正常/无结果/边界情况

**Facet 服务关系**：
- 结构化示例 → 规范型 facet（输出格式）+ 知识型 facet（输入格式）
- 自然语言示例 → 规范型 facet（NL 输出）+ 边界型 facet（边界处理）

**记住**：结构化示例展示结构，自然语言示例展示语气。两者都不是让模型照搬。
