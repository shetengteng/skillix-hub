## 精炼核心记忆（必须执行）

请对 MEMORY.md 进行精炼优化。读取 {memory_md_path} 的当前内容，综合本次会话的新事实，生成精炼后的完整记忆内容。

### 核心判断标准

**只保留"下个月还重要"的信息**。对每条记忆问自己：这条信息在 30 天后的新会话中，是否仍然有助于理解用户/项目？

写入 MEMORY.md 的信息：
- 用户明确表达的偏好（如"中文回答"、"偏好 TypeScript"）
- 项目级持久上下文（如"数据库用 PostgreSQL"、"项目采用微服务架构"）
- 重要的架构决策（如"选择方案 A 而非方案 B"）
- 用户明确说"记住这个"的内容

不写入 MEMORY.md 的信息：
- 一次性任务的具体步骤和过程
- 临时性数量统计（测试通过率、文件数、数据条数）
- 已完成任务的进度描述
- Bug 修复的具体步骤（只保留结论性规范）
- 闲聊和无持续价值的信息

### 大小约束

MEMORY.md 应保持精简，**总条目不超过 50 条**（约 2000 字）。超出时优先删除：
- 已过时的项目背景（如已废弃的技术栈）
- 已完成且不再影响后续开发的决策
- 可从代码中直接推断的信息

**永远保留**：用户偏好、项目规范。

如果本次会话没有产生新的重要事实，且现有 MEMORY.md 内容仍然准确，可以跳过精炼。

### 精炼规则

1. 每条记忆不超过 80 字，使用结论性表述
2. 合并语义重复的条目（如同一事件的多次描述）
3. 过滤临时性信息（数量统计、进度报告、已修复 Bug 的具体步骤）
4. 准确分类到：用户偏好 / 项目规范 / 重要决策 / 项目背景 / 常用工具
5. 保留所有仍然有效的现有记忆，只精炼过长的条目
6. 删除已过时或被后续事实更新的旧信息

### 执行方式

调用精炼写入工具，传入精炼后的完整 MEMORY.md 内容（JSON 格式，每个章节一个 key，value 为条目数组）：

```bash
{distill_cmd} --project-path "{project_path}" --session "{conv_id}" --content '{{"用户偏好":["条目1","条目2"],"项目背景":["条目1"],"项目规范":["条目1"],"重要决策":["条目1"],"常用工具":["条目1"]}}'
```
